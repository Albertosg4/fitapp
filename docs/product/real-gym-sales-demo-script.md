# Guion de demo comercial — Gimnasio

## Objetivo de la demo

- Mostrar cómo el gimnasio gestiona actividades, horarios, socios, reservas, pagos y QR.
- No hablar de multi-sector todavía.
- No enseñar Supabase, GitHub ni código.
- No prometer multi-tenant real todavía.

## Orden recomendado de demo

1. Presentar app.
2. Enseñar admin.
3. Enseñar actividades.
4. Enseñar horarios/clases.
5. Enseñar socios.
6. Enseñar pagos.
7. Enseñar área socio.
8. Reservar/cancelar clase.
9. Enseñar QR/check-in.
10. Cerrar con próximos pasos.

## Frases recomendadas

- "Esta es la vista del gimnasio para gestionar socios, clases y reservas."
- "El socio puede reservar desde su móvil sin llamar ni escribir por WhatsApp."
- "El QR sirve para controlar accesos y asistencia."
- "La parte de pagos permite centralizar renovaciones y membresías."
- "La demo multi-sector existe, pero esta vista está enfocada en gimnasio."

## Cosas que NO decir todavía

- "Esto ya sirve para cualquier sector."
- "Es multi-tenant completo."
- "Podemos migrar cualquier clínica mañana."
- "Los pagos están auditados al 100%."
- "Tenemos cuenta demo persistida para todos los casos."
- "Se puede borrar/resetear datos sin riesgo."

## Preguntas para el cliente

- ¿Cuántos socios tiene el gimnasio?
- ¿Cuántas clases semanales?
- ¿Cómo gestionan reservas ahora?
- ¿Usan pagos online?
- ¿Necesitan control de acceso?
- ¿Tienen varios centros?
- ¿Qué datos necesitan ver en el panel?
- ¿Qué les genera más trabajo manual?
